# Используется: после открытого вопроса (Шаг 3)
# Цель: понять чего не хватает для хорошего отзыва
# Модель: Azure OpenAI GPT-4o
# Важно: дешёвый запрос — короткий ответ, только JSON

Тебе дан ответ человека о работе с Сергеем Хабаровым.

Проверь, есть ли в ответе следующие элементы:

**context** — понятно кто пишет и в каком качестве работали с Сергеем.
Примеры того что засчитывается: "проходил курс", "работали над проектом", "консультировался", "общались на конференции".

**moment** — есть конкретный момент, результат или что-то изменилось.
Примеры того что засчитывается: конкретное событие, результат после работы, что применил, что запомнилось.
Общие фразы не засчитываются: "всё понравилось", "было полезно", "хороший человек".

**style** — есть характеристика того как Сергей работает или чем отличается.
Примеры того что засчитывается: "объясняет просто", "всегда в теме", "не давит", "быстро схватывает суть".
Общие оценки не засчитываются: "профессионал", "хороший специалист", "умный".

---

Верни JSON и только JSON. Без пояснений, без markdown, без текста до или после.

```json
{
  "context": true,
  "moment": false,
  "style": true,
  "enough": false
}
```

**Правило для `enough`:**
`enough = true` если заполнены хотя бы два из трёх полей, и одно из них обязательно `moment`.
Если `moment = false` — `enough` всегда `false`, даже если остальные заполнены.
